MiniJava Compiler & Interpreter

A clean, self-contained compiler for the MiniJava language — built from scratch, zero dependencies, fully tested.

Java 8+ No dependencies 6 source files 6 test programs Tree-walking interpreter

Overview

This is a fully working compiler and interpreter for MiniJava — a clean subset of Java designed for teaching compiler construction. It takes a .java source file written in MiniJava syntax, checks it for errors, and executes it.

The compiler implements every classical phase of a real compiler:

Source (.java)
→
Lexer
→
Parser
→
AST
→
Type Checker
→
Interpreter
→
Output
PhaseWhat it doesFile
LexerConverts raw text into a stream of tokens (keywords, identifiers, operators)Lexer.java
ParserReads tokens and builds an Abstract Syntax Tree (AST) using recursive descentParser.java
Type CheckerBuilds a symbol table, checks all types, validates method calls and variable usesTypeChecker.java
InterpreterWalks the AST and executes it — runs classes, methods, loops, arrays, and inheritanceInterpreter.java

Installation

Requirements

You only need a Java Runtime Environment (JRE) or JDK version 8 or higher. No build tools, no Maven, no Gradle, no extra libraries.

ℹ
To check your Java version, run java -version in a terminal. Any output showing version 8, 11, 17, 21, or higher is fine.

Step 1 — Unzip the Package

bash / terminal 
unzip MiniJavaCompiler.zip
cd MiniJavaCompiler

Step 2 — Compile the Compiler

Run the build script for your platform:

Linux / macOS 
bash build.sh
Windows 
build.bat

Or manually:

Any platform 
javac -d bin src/*.java
✔
You should see: Build successful! Compiler is in bin/ — the bin/ folder will now contain all compiled .class files.

Compiler Modes

The compiler has four modes, selected with a flag before the filename:

FlagModeWhat it shows
(none)RunCompiles and runs the program, printing output to console
--lexLexRuns only the lexer and prints all tokens with their types and positions
--parseParseRuns lexer + parser and prints a summary of the AST structure
--checkType CheckRuns lexer + parser + type checker and reports any type errors

Error Messages

The compiler produces clear, line-numbered error messages for three categories of problems:

Lexer Errors

Triggered by characters the lexer doesn't recognise.

example 
[MiniJava Error] Lexer error at line 5, col 12: unexpected character '@'

Parser Errors

Triggered by syntax that doesn't match MiniJava grammar rules.

example 
[MiniJava Error] Parse error at line 8: expected SEMICOLON but got RBRACE ("}")

Type Errors

Triggered by type mismatches, undeclared variables, unknown classes, wrong argument counts, etc.

example 
[MiniJava Error] Type error at line 9: Arithmetic requires int operands at line 9
⚠
All error messages are printed to stderr. If you are piping output to a file, use 2>&1 to capture errors too: java -cp bin MiniJava prog.java > out.txt 2>&1

Syntax Reference

MiniJava is a strict subset of Java. A valid MiniJava program has exactly one main class followed by zero or more regular classes.

Program Structure

MiniJava 
class Main {
    public static void main(String[] args) {
        // statements here
        System.out.println(42);
    }
}

class MyClass {
    // field declarations
    int x;

    // method declarations
    public int getX() {
        return x;
    }
}

Statements

Print

syntax 
System.out.println(expr);

Prints the integer or boolean value of expr followed by a newline.

Assignment

syntax 
identifier = expr;

The variable must be declared in the current method, as a parameter, or as a field of this.

Array Assignment

syntax 
array[index] = expr;

If / Else

syntax 
if (cond) statement
if (cond) statement else statement

Condition must be a boolean. The else branch is optional.

While Loop

syntax 
while (cond) statement

Block

syntax 
{ statement statement ... }

Groups multiple statements. Often used with if/else and while.

Classes & Methods

Declaring a Class

MiniJava 
class Animal {
    int legs;           // field

    public int getLegs() {
        return legs;
    }
}

Inheritance

MiniJava 
class Dog extends Animal {
    public int speak() {
        return 1;   // 1 means "woof"
    }
}

A subclass inherits all fields and methods from its superclass. Method overriding is supported.

Creating Objects

MiniJava 
Dog d;
d = new Dog();
System.out.println(d.speak());

Method Rules

  • All methods must be public
  • Every method must end with a return statement
  • Variable declarations must come before statements in method bodies
  • Methods can call other methods on this or on other objects
  • Recursion is fully supported

Arrays

MiniJava 
int[] arr;
arr = new int[10];    // allocate 10 elements (all zero)
arr[0] = 42;         // set element
System.out.println(arr[0]);   // read element
System.out.println(arr.length); // get length

Sample Programs

1. Factorial — Recursion

test/Factorial.java  → output: 3628800 
class Factorial {
    public static void main(String[] a) {
        System.out.println(new Fac().compute(10));
    }
}
class Fac {
    public int compute(int n) {
        int result;
        if (n < 1)
            result = 1;
        else
            result = n * (this.compute(n - 1));
        return result;
    }
}

2. Fibonacci — Multiple Recursion

test/Fibonacci.java  → output: 0, 1, 55 
class Fib {
    public int compute(int n) {
        int result;
        if (n < 2)
            result = n;
        else
            result = this.compute(n - 1) + this.compute(n - 2);
        return result;
    }
}

3. BubbleSort — Arrays & Nested Loops

test/BubbleSort.java  → output: 1 2 3 4 5

Fills an array with [5,4,3,2,1] and sorts it in-place using bubble sort, then prints each element.

4. Inheritance — extends & Override

test/Inheritance.java  → output: 1, 2, 4

Dog and Cat both extend Animal. Each overrides speak() and inherits legs().

5. LinearSearch — Boolean Logic

test/LinearSearch.java  → output: 1, 0

Searches an array [0, 3, 6, 9, ...] for a target value, returns 1 if found, 0 if not.

6. Counter — Object Fields

test/Counter.java  → output: 5, 0

Demonstrates reading and writing object fields across multiple method calls on the same object.

Limitations

MiniJava is intentionally a subset of Java. The following Java features are not supported:

FeatureStatusWorkaround
for loopNot supportedUse while loop
i++ / i--Not supportedUse i = i + 1
String variablesNot supportedUse int codes
float / doubleNot supportedUse integer arithmetic
Multiple classes per fileSupportedAll classes in one file
null valueNot supportedUse 0 or a sentinel int
Static methods (besides main)Not supportedUse instance methods
Multiple inheritanceNot supportedSingle extends only
> >= <= ||Not supportedFlip operands or use ! + &&
Interfaces / abstractNot supportedUse concrete classes

Architecture

Compiler Pipeline

The compiler is structured as a clean linear pipeline. Each phase reads from the previous phase's output and produces a new representation:

PhaseInputOutputKey class
Lexical AnalysisRaw source stringList<Token>Lexer
ParsingList<Token>AST.ProgramParser
Type CheckingAST.ProgramValidated (side-effect)TypeChecker
InterpretationAST.ProgramProgram output (stdout)Interpreter

Design Patterns Used

  • Recursive Descent Parsing — Each grammar rule maps directly to a method in Parser.java
  • Discriminated Union via Inheritance — All AST nodes are inner classes of AST.java, using instanceof dispatch
  • Environment Map — The interpreter uses Map<String, Object> as a call-stack frame, one per method invocation
  • Superclass Chain Lookup — Both the type checker and interpreter walk the extends chain for method and field resolution
